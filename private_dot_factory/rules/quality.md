# Quality Rules

- No hacks, monkey patches, fake fixes, or temporary workarounds.
- Fix root causes instead of masking symptoms.
- Prefer the smallest clean solution that preserves maintainability.
- Inspect existing code before introducing conventions.
- Touch only files directly related to the task.
- Do not refactor unrelated code.
- Report uncertainty honestly.
- Validate with the narrowest relevant command before claiming success.
